## ❓ Frequently Asked Questions (FAQ)

**Q1: What is Celtrix?**  
A1: Celtrix is a powerful project scaffolding tool that helps developers quickly generate full-stack applications with clean folder structures and modern best practices. It saves hours of setup time by automating the creation of ready-to-use templates.

**Q2: Why was Celtrix created?**
A2: Developers often spend a lot of time setting up the same boilerplate code for every new project. Celtrix was built to automate that process and let you focus on writing actual business logic instead of repetitive configuration.

**Q3: Who is Celtrix for?**
A3: Celtrix is ideal for beginners, students, and professional developers who frequently start new projects, contribute to hackathons, or need consistent and reliable setups for different stacks.

**Q4: How do I get started with Celtrix?**
A4: Getting started is simple! You don’t need to install anything globally. Just run:

npx celtrix my-awesome-app


Follow the prompts, choose your preferred stack, and Celtrix will create your project automatically.

Q5: What makes Celtrix stand out from other project scaffolding tools?**
A5: Celtrix provides ready-to-use templates, supports multiple popular stacks, and follows modern best practices, allowing you to start projects quickly without manual setup.

**Q6: Which stacks are currently available in Celtrix?**
A6: Celtrix currently supports MERN, MEAN, T3, Angular+Tailwind, and several other popular stacks. More stacks are regularly added by contributors.

**Q7: Does Celtrix support TypeScript projects?**
A7: Yes! Most templates include optional TypeScript support that you can enable when generating a new project.

**Q8: Are Celtrix projects production-ready?**
A8: Celtrix projects provide a strong foundation for development. Some production-specific adjustments, such as deployment configs or CI/CD setup, may be needed depending on your project.

**Q9: Can I customize the generated templates?**
A9: Absolutely! You can modify components, APIs, styles, and configurations to match your project’s unique requirements.

**Q10: How do I add a new stack or template to Celtrix?**
A10: Create a new template in the templates/ folder following the existing structure, test it with the CLI, and then open a Pull Request to contribute your stack.

**Q11: How can I improve or update existing templates?**
A11: Fork the repository, create a new branch for your changes, test them thoroughly, and submit a Pull Request. Make sure your updates follow the project’s coding guidelines.

**Q12: How can I report bugs or request new features?**
A12: Open a GitHub Issue in the Celtrix repository with detailed steps, screenshots, or examples to help maintainers understand and resolve your issue.

**Q13: What prerequisites do I need to use Celtrix?**
A13: You only need Node.js and npm installed. Celtrix automatically installs and manages all other dependencies during setup.

**Q14: Can I use Celtrix for commercial projects?**
A14: Yes! Celtrix is open-source under the ISC License, allowing both personal and commercial use.

**Q15: Is Celtrix still in active development?**
A15: Yes, Celtrix is actively maintained and open to community contributions. New features, templates, and improvements are added regularly.

**Q16: How can I contribute if I’m new to open source?**
A16: You can start small — fix typos, improve docs, or test templates. Every contribution counts! Check the Contributing Guide for detailed steps.

**Q17: Where can I get support or connect with the community?**
A17: You can ask questions in GitHub Discussions or join the community chat (if available). The Celtrix community is always happy to help new contributors.
